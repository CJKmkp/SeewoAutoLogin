# 希沃自动登录

ICC-CE 插件，用于管理希沃账号，并为希沃白板提供快捷登录能力。

## 功能

- 支持多个希沃账号管理。
- 支持账号密码登录并保存账号信息。
- 支持希沃官方二维码登录。
- 二维码登录会从希沃账号中心获取短时会话，使用希沃官方移动端扫码并确认登录。
- 支持选择当前活动账号。
- 支持通过本地 SSO 网关向希沃白板提供账号列表和登录令牌。
- 支持扫码账号会话恢复和每日令牌刷新。
- 支持用户列表轮换：可按 4 个或 6 个账号分组，在希沃快捷登录窗口短时间重新打开时切换账号组。
- 支持为插件设置页增加独立密码保护。
- 支持中英文界面文案。

## 使用方法

### 添加账号

1. 在 ICC-CE 插件管理器中安装插件。
2. 打开插件设置页。
3. 点击“添加账号”。
4. 选择一种方式：
   - **账号密码登录**：填写显示名称、手机号/用户名和密码，点击“登录并保存”。
   - **官方扫码登录**：点击“开始扫码登录”，使用希沃官方移动端扫描二维码，并在手机上确认。
5. 登录成功后，账号会加入账号列表。
6. 在账号列表中选择要使用的活动账号。

### 二维码登录

二维码登录流程如下：

1. 插件连接 `id.seewo.com` 获取短时二维码会话。
2. 使用希沃官方移动端扫描二维码。
3. 在手机端确认登录。
4. 插件获取账号资料并保存会话凭据。
5. 后续启动时尝试恢复会话；可用时每日自动刷新令牌。

如果二维码过期、网络异常或手机端拒绝登录，可以点击刷新二维码后重试。

### 密码保护

在“安全设置”中启用“使用密码保护插件设置”，并设置插件独立密码。启用后，进入插件设置页需要先验证密码。

该密码只用于保护本插件设置页，不等同于希沃账号密码，也不会上传到希沃服务器。

### 用户列表轮换

用户列表轮换默认关闭。启用后可以选择每组 4 个或 6 个用户。

当希沃快捷登录窗口关闭后，在约 10 秒内再次打开时，插件会切换到下一组账号列表。该功能用于账号较多时减少快捷登录窗口中的账号数量。

## 数据与安全

- 账号配置保存在 ICC-CE 的插件配置目录中：

  ```text
  PluginConfigs/com.icc.seewo-autologin/config.json
  ```

- 扫码登录令牌使用插件的本地会话存储机制保存，不写入诊断日志。
- 插件诊断日志会记录流程状态和错误类型，但不会记录二维码会话密钥或令牌。
- 账号密码登录所需的网络请求由插件直接发送到希沃账号服务。
- 请只在可信设备上使用本插件，并避免将插件配置文件和日志文件发送给他人。
- 删除账号时，插件会同时尝试删除对应的本地扫码会话凭据。

## 权限

插件声明以下权限：

- `Network`：访问希沃账号服务，完成密码登录、二维码登录和令牌刷新。
- `Settings`：保存账号、活动账号、轮换和密码保护设置。

## 常见问题

### 二维码无法加载

请检查网络连接、防火墙和代理设置，确认可以访问希沃账号服务，然后点击“刷新二维码”。

### 扫码后没有完成登录

请确认使用的是希沃官方移动端，并且已经在手机端确认授权。二维码过期后需要刷新，不能继续使用旧二维码。

### 重启后账号仍在，但无法自动登录

扫码会话可能已过期或本地凭据不可用。重新执行官方扫码登录即可更新会话。

### 为什么看不到所有账号

如果启用了用户列表轮换，希沃快捷登录窗口会按配置的分组大小显示账号。重新打开窗口可在轮换窗口内切换到下一组。

## 构建

需要 Windows 和 .NET 6 SDK：

```powershell
dotnet restore SeewoAutoLogin.csproj
dotnet build SeewoAutoLogin.csproj -c Release
```

GitHub Actions 支持推送 `v*` 标签或手动运行工作流，并自动构建、打包和发布 `.icpx` 文件。

## 发布包安装

从 GitHub Release 下载：

```text
com.icc.seewo-autologin.icpx
```

将文件复制到：

```text
PluginPackages/
```

然后重启 ICC-CE，插件管理器会自动处理安装。
